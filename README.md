# young-s-library (P-Reinforce Knowledge Agent)

**young-s-library**는 지식의 중력을 거스르는 'P-Reinforce' 엔진 기반의 자동화 영속적 위키(Permanent Wiki) 시스템입니다. 사용자의 원시(Raw) 메모와 데이터를 입력받아 의미 있는 지식으로 자동 변환하고 노드들을 유기적으로 연결하여 거대한 '외부 뇌'를 구축합니다.

## 🚀 Core Features

- **RL 기반 자동 구조화**: 강화학습(Reinforcement Learning) 보상 정책에 따라 지식을 분석하고, 유사도에 맞추어 스스로 최적의 폴더를 생성하거나 분류합니다.
- **영구적인 지식 그래프**: 파편화된 정보를 단발성으로 두지 않고, 쌍방향 링크(`[[문서]]`)로 엮어 끊임없이 성장하는 지식 생태계를 만듭니다.
- **동적 폴더링 (Auto-Refactoring)**: 특정 그룹 내 문서가 많아질 경우(기본 12개 초과) 하위 개념을 도출하여 자체적으로 구조를 세분화합니다.
- **Git 자동 동기화**: 모든 지식의 추가, 이동, 구조 리팩토링 내역은 시간순으로 GitHub에 커밋(Commit) 및 Push되어 사고의 타임라인으로 영구 보존됩니다.

## 📂 Structure

- `00_Raw/` : 가공되지 않은 사용자의 원본 데이터가 날짜별로 누적되는 공간 (Source of Truth)
- `10_Wiki/` : P-Reinforce 엔진이 구조화된 문서를 Projects, Topics, Decisions, Skills 등으로 자동 배치하는 핵심 지식층
- `20_Meta/` : 지식 엔진의 상태를 나타내는 데이터 (`Graph.json`, `Policy.md`, 전체 목차 `Index.md`)
- `.github/` : Git 동기화 및 에이전트 워크플로우를 담당하는 설정 계층

## 💡 How It Works

1. 사용자가 `00_Raw/` 위치에 자유롭게 아이디어나 텍스트를 저장합니다.
2. P-Reinforce 에이전트가 이를 모니터링하여 **통찰 요약이 포함된 표준 규격(탬플릿)** 으로 내용을 정제합니다.
3. 문서의 맥락을 파악해 `10_Wiki/` 하위에 위치시키고, 관련된 기존 문서나 폴더를 찾아서 자동 링크합니다.
4. 모든 변화를 사용자 대신 자동 커밋(`git add . & git commit ...`)하여 GitHub에 동기화합니다.

## 🧠 Feedback Loop

에이전트는 사용자의 작은 피드백에도 반응하여 분류 체계를 지속적으로 학습합니다:
- **수용 (칭찬)**: "이 분류 완벽해"라는 인정이나, 분류 결과를 그대로 사용하는 행위 자체가 가중치를 강화시킵니다.
- **수정 (이동)**: 문서가 잘못 분류되어 사용자가 직접 위치를 수정하면, 에이전트는 두 카테고리 간의 경계(Boundary vector)를 재설정하여 차후 판단에 반영합니다.
